### Core Understanding of MIP

A **Valid MIP** is created when:
- The original problem has a UNIQUE, DETERMINATE solution
- By removing specific conditions, it becomes UNSOLVABLE/UNDETERMINABLE (not "infinite solutions")
- The problem logic breaks because a necessary link is missing
- The solver cannot derive the answer and must state "insufficient information"

An **Invalid MIP** occurs when:
- Removing conditions makes the problem OPEN-ENDED with infinitely many valid answers
- Or mathematical errors are introduced during rewriting
- Or essential assumptions/parameters are deleted

---

### Verification Criteria

Check if the rewritten problem has any of these THREE CRITICAL ISSUES:

---

**Issue 1: From Uniquely Solvable to Unsolvable (VALID) vs. To Infinitely Solvable (INVALID)**

The key distinction:
- ✅ **VALID**: Original is uniquely solvable → Rewrite becomes unsolvable (missing dependency)
- ❌ **INVALID**: Original is uniquely solvable → Rewrite becomes infinitely solvable (lost constraints)

**✅ Valid MIP Examples (Uniquely Solvable → Unsolvable):**
1. Original: "Given x = 2 and xy = 10, find y"
   Rewrite: "Given xy = 10, find y"
   → VALID: Cannot determine y without knowing x (unsolvable logic)

2. Original: "Circle with center (0,0) and radius 5"
   Rewrite: "Circle with center (0,0)"
   → VALID: Cannot determine the specific circle without radius (missing parameter)

3. Original: "If a = 3 and b = 4, find a² + b²"
   Rewrite: "If a = 3, find a² + b²"
   → VALID: Cannot compute without knowing b (missing variable)

**❌ Invalid MIP Examples (Uniquely Solvable → Infinitely Solvable):**
1. Original: "Find THE LARGEST prime divisor of 60"
   Invalid Rewrite: "Find a prime divisor of 60"
   → INVALID: Now 2, 3, 5 are all valid answers (lost uniqueness constraint)

2. Original: "Find tan(x) where tan(53)tan(81)tan(x) = tan(53) + tan(81) + tan(x), and 0 < x < 180"
   Invalid Rewrite: Removed "0 < x < 180"
   → INVALID: Now x = 7, 187, 367, ... all work due to periodicity (infinite solutions)

3. Original: "Find the integer n where n² < 50 < (n+1)²"
   Invalid Rewrite: "Find n where n² < 50 < (n+1)²"
   → INVALID: Now n can be 7, 7.1, 7.5, ... infinitely many real numbers work (lost integer constraint)

**Detection Rule:**
1. Does removing the condition break a DEPENDENCY (making it unsolvable)? → VALID
2. Or does it remove a CONSTRAINT (allowing infinite arbitrary answers)? → INVALID

---

**Issue 2: Mathematical Errors Introduced**

The rewriting process must NEVER change numbers, flip signs, or create illegal operations.

**Critical Rules:**
- Numbers must NEVER be changed: 0.25 cannot become -0.25
- Signs must NEVER be flipped: log(x) cannot become log(-x)
- Operations must remain valid: sqrt(x) requires x ≥ 0

**❌ Examples of Mathematical Errors:**
1. Changed "log(0.25)" to "log(-0.25)"
   → INVALID: Introduced undefined operation (log of negative number)

2. Changed "x > 0" to "x < 0" in a problem with sqrt(x)
   → INVALID: Makes sqrt(x) undefined for the domain

3. Changed "sin(30°)" to "sin(-30°)"
   → INVALID: Flipped the sign, altering the mathematical value

**Detection Rule:**
- Compare all numbers, signs, and operations between original and rewrite
- ANY difference → INVALID

---

**Issue 3: Assumptions or Fixed Parameters Deleted**

Fixed assumptions and parameters (using "Assume", "Let", "Suppose") are NOT removable conditions. They are essential context that defines the problem setup.

**❌ Invalid Example (Assumption Deleted):**
- Original: "A firecracker explodes into pieces. Find the speed of piece A. Assume g = 10 m/s²."
- Invalid Rewrite: "A firecracker explodes into pieces. Find the speed of piece A."
- Why Invalid: "g = 10" is a fixed parameter assumption, not a variable condition

**Trigger Phrases to Watch:**
- "Assume", "Suppose", "Let's say", "Given that"
- "For example", "For instance", "e.g.", "such as"
- "Note that", "It is known that"

**Detection Rule:**
- If the original contains assumption phrases and they're removed → INVALID
- These provide essential context and cannot be treated as removable conditions

---

### Verification Steps

**Step 1: Check Issue 1 (Uniquely Solvable → Unsolvable vs. Infinitely Solvable)**
- Does removing the condition break a dependency (cannot derive answer)? → VALID
- Or does it remove a uniqueness constraint (infinite answers now valid)? → INVALID

**Step 2: Check Issue 2 (Mathematical Errors)**
- Are all numbers, signs, and operations identical between original and rewrite?
- Any difference → INVALID

**Step 3: Check Issue 3 (Assumptions Deleted)**
- Does the original contain "Assume", "For example", or similar phrases?
- Are they preserved in the rewrite? If not → INVALID
